# 소노 GS팀 대시보드 프로젝트 트래커

## 세션 멈춤 원인 분석 & 리스크 리포트

### 원인 분석

| 원인 | 설명 | 영향도 |
|------|------|--------|
| 컨텍스트 윈도우 초과 | 세션이 100턴 이상 진행되면 모델의 컨텍스트 윈도우가 포화. 패키지 분류(127턴), 메인기사 아이콘(137턴)에서 발생 | 높음 |
| 대용량 파일 처리 | palatium.html(786KB), index.html 등 대형 HTML을 읽고 수정하면 컨텍스트 급속 소모 | 높음 |
| 복합 작업 단일 세션 할당 | 파싱+빌드+UI수정+커밋을 하나의 세션에서 처리하면 턴 수 폭증 | 중간 |
| 워크트리 경로 불일치 | 파일이 다른 프로젝트 폴더에 있을 때 세션이 잘못된 경로 탐색에 턴 소비 | 낮음 |
| 병렬 세션 리소스 경합 | 동시 실행 세션이 많으면 개별 세션 응답 지연 | 낮음 |

### 대응 방안

| 방안 | 설명 |
|------|------|
| 작업 분할 | 하나의 큰 작업을 파싱→빌드→커밋 등 단계별로 분리하여 각각 별도 세션 |
| 50턴 이하 유지 | 세션당 턴 수를 50 이하로 유지, 초과 시 새 세션으로 이관 |
| 파일 직접 수정 최소화 | 대형 HTML 전체 읽기 대신 특정 섹션만 grep/sed로 수정 |
| 경로 사전 확인 | 작업 시작 전 파일 위치를 먼저 확인하고 세션에 정확한 경로 전달 |
| idle 세션 정리 | 완료된 idle 세션의 워크트리 주기적 정리로 디스크 공간 확보 |

### 현재 리스크 항목

| 리스크 | 상태 | 심각도 | 대응 |
|--------|------|--------|------|
| 팔라티움 DB 경로 혼선 | 해결 중 | 중 | gs_daily_trend_news_public_temp/data/palatium_db/로 확인 완료 |
| 자동화 스크립트 미머지 | 대기 | 중 | refresh_dashboards.py + LaunchAgent가 워크트리에만 존재 |
| 온북 전체 페이지 BI 교체 미완 | 대기 | 높 | index.html 전체 페이지 툴팁/연도필터 적용 필요 |
| 다올 PMS 연동 불가 | 보류 | 낮 | 윈도우 보안정책으로 Python 불가, 수기 전달 방식 유지 |
| 자사몰 크롤링 보류 | 보류 | 낮 | API 불안정, FIT 요금표로 대체 |

## 파일 구조

### sono-competitor-crawler (경쟁사 크롤링 + 팔라티움)
```
scripts/
  parse_palatium.py        # 팔라티움 Excel 파서
  build_palatium.py        # palatium JSON→HTML
  refresh_dashboards.py    # 통합 갱신 (온북+팔라티움)
  crawler.py               # OTA 크롤링 (야놀자/트립닷컴/네이버/브랜드몰)
data/
  palatium_data.json       # 팔라티움 파싱 결과
  palatium_db/             # 다올 PMS 엑셀 원본
  fit_rates_source.xlsx    # FIT 요금표
  2026_budget.xlsx         # 사업계획
docs/
  index.html               # 경쟁사 모니터링 페이지
  palatium.html            # 팔라티움 대시보드
config.yaml                # 크롤링 설정 (로그인 포함)
```

### gs_daily_trend_news_public_temp (온북 트렌드 리포트)
```
scripts/
  parse_raw_db.py          # 온북 txt DB 파서
  db_to_notes.py           # DB→daily_notes.json (BI 대체)
  build.py                 # HTML 빌드
  parse_package_trend.py   # 패키지 분류 집계
  generate_insights.py     # 뉴스 인사이트 생성
data/
  raw_db/                  # 온북 txt 원본 (27/28/43/44)
  db_aggregated.json       # 파싱 결과
  daily_notes.json         # 빌드 입력
  package_series_trend.json # 패키지 TOP 100 집계
  palatium_db/             # (팔라티움 DB도 여기에 복사)
docs/
  index.html               # 온북 트렌드 메인
  otb.html                 # OTB 대시보드
  data/otb_data.json       # OTB 피벗 데이터
.github/workflows/
  deploy.yml               # GitHub Actions 자동 배포
```
